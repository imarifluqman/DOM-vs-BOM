DOM — Document Object Model

The Document as a Living Tree

When a browser loads HTML, it parses the markup and builds a tree of objects in memory. This tree is the DOM — and JavaScript can read and rewrite every node of it.

DOM Tree Structure
window  ← global object (BOM)
└─ document  ← entry point to DOM
   └─ <html>
      ├─ <head>
      │  └─ <title>  text node inside
      └─ <body>
         ├─ <div id="app">
         │  ├─ <h1>  Element node
         │  └─ <p>  Element node
         └─ <button>  Element node
dom-examples.js
// ── SELECTING ELEMENTS ─────────────────────────────
const byId    = document.getElementById('app')
const byQuery = document.querySelector('.btn')
const allBtns = document.querySelectorAll('button')
const byTag   = document.getElementsByTagName('p')

// ── READING & MODIFYING CONTENT ────────────────────
element.innerHTML   = '<strong>Hello</strong>'
element.textContent = 'Plain text only'
element.setAttribute('data-id', '42')
element.getAttribute('href')

// ── CREATING & INSERTING NODES ─────────────────────
const div = document.createElement('div')
div.className   = 'card'
div.textContent = 'Dynamic content!'
document.body.appendChild(div)
document.body.removeChild(div)

// ── STYLING ────────────────────────────────────────
element.style.color   = '#00e5ff'
element.style.display = 'none'
element.classList.add('active')
element.classList.toggle('visible')

// ── EVENTS ─────────────────────────────────────────
element.addEventListener('click', (e) => {
  console.log('Clicked!', e.target)
})

// ── TRAVERSAL ──────────────────────────────────────
element.parentNode
element.children
element.firstElementChild
element.nextElementSibling
querySelector()
Returns the first element matching a CSS selector. Most versatile selector method.
document.querySelector('#nav .link')
createElement()
Creates a new HTML element node that can be inserted into the DOM.
const el = document.createElement('div')
addEventListener()
Attaches an event handler for click, keyup, scroll, and hundreds of other events.
el.addEventListener('click', handler)
classList API
Add, remove, toggle, and check CSS classes on any element cleanly.
el.classList.toggle('active')
innerHTML / textContent
Get or set the HTML or plain-text content of an element.
el.textContent = 'Hello world'
dataset
Access custom data-* attributes as a DOMStringMap on any element.
el.dataset.userId // data-user-id
BOM — Browser Object Model

The Browser as Your API

Beyond the document lies the browser itself. The BOM gives JavaScript access to navigation, history, screen data, timers, storage — all hanging off the global window object.

BOM Object Hierarchy (all on window)
window  ← the global scope itself
├─ location  ← URL, pathname, hash, redirect
├─ history  ← back(), forward(), pushState()
├─ navigator  ← userAgent, language, online, geolocation
├─ screen  ← width, height, colorDepth
├─ localStorage  ← persistent key-value storage
├─ sessionStorage  ← tab-scoped storage
├─ setTimeout / setInterval  ← timer APIs
├─ fetch / XMLHttpRequest  ← network requests
└─ document  ← the entire DOM lives here too
bom-examples.js
// ── window.location ────────────────────────────────
location.href        // full URL string
location.hostname    // e.g. "example.com"
location.pathname    // e.g. "/products/shoes"
location.search      // e.g. "?color=red&size=10"
location.hash        // e.g. "#section-2"
location.assign('https://google.com')  // navigate
location.reload()                      // refresh page

// ── window.history ─────────────────────────────────
history.back()
history.forward()
history.go(-2)        // go 2 pages back
history.pushState({}, '', '/new-url')   // SPA routing
history.replaceState({}, '', '/updated') // no new entry

// ── window.navigator ───────────────────────────────
navigator.userAgent      // browser/OS string
navigator.language       // e.g. "en-US"
navigator.onLine         // true / false
navigator.cookieEnabled
navigator.geolocation.getCurrentPosition(pos => {
  console.log(pos.coords.latitude, pos.coords.longitude)
})

// ── window.screen ──────────────────────────────────
screen.width        // physical screen width
screen.height
window.innerWidth   // viewport width (more useful)
window.innerHeight

// ── Timers ─────────────────────────────────────────
const id = setTimeout(() => console.log('delayed'), 2000)
clearTimeout(id)
const iid = setInterval(() => console.log('tick'), 1000)
clearInterval(iid)

// ── Storage ────────────────────────────────────────
localStorage.setItem('theme', 'dark')
localStorage.getItem('theme')      // → 'dark'
localStorage.removeItem('theme')
sessionStorage.setItem('token', 'abc')

// ── Dialogs ────────────────────────────────────────
alert('Hello!')
const ok  = confirm('Are you sure?')  // → boolean
const val = prompt('Enter name:')      // → string
window.location
Full URL access and navigation control. Essential for SPAs and redirect logic.
location.pathname // '/about'
window.history
Navigate browser history and manipulate URL without page reload via pushState.
history.pushState({}, '', '/new')
window.navigator
Device and browser info — user agent, language, online status, geolocation.
navigator.language // 'en-US'
localStorage
Persistent key-value storage that survives page refreshes and browser restarts.
localStorage.setItem('k', 'v')
setTimeout / setInterval
Schedule code to run once after a delay, or repeatedly at a given interval.
setTimeout(fn, 1000)
window.screen
Physical display dimensions and color depth of the user's monitor.
screen.width + 'x' + screen.height

DOM vs BOM

A direct comparison across the dimensions that matter most.

Property DOM BOM
Full Name Document Object Model Browser Object Model
Root Object document window
Represents HTML document as a node tree Browser environment around the document
Standard ✅ W3C / WHATWG formal spec ⚠️ No formal standard — browser-specific
Cross-browser Highly consistent Mostly consistent, some quirks
Key Objects document, Element, Node, Event location, history, navigator, screen
Primary Use Manipulate page content & structure Navigation, storage, timers, environment
Node.js? Via jsdom library ❌ Browser-only
Relationship document lives inside window window wraps and contains document
